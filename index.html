Спецификация Alfa API

Введение

Это API использует протоколы oAuth 2.0, OIDC. Позволяет внешнему приложению обращаться к программным системам внутри самого Банка.

Alfa API - это сервис безопасной интеграции банковских продуктов и услуг в бизнес-процессы Партнеров API.

Alfa ID - универсальная учетная запись, обеспечивающая единый вход Клиентов на разные ресурсы Банка и Партнера API, в том числе, предоставляющая подписание Клиентами юридически значимых согласий и двухсторонний обмен данными о Клиентах (персональные данные, платежные данные) между Банком и Партнерами API.

Партнер API – Клиент, заключивший с Банком договор на информационно-технологическое взаимодействие, в целях использования Сервиса Alfa API.BaaS, по утвержденной Банком форме.

Подключение к Alfa API

Для подключения к Alfa API необходимо выполнить действия:

Шаг 1. Стать клиентом «Альфа-Банка», оформить расчетно-кассовое обслуживание и подключить «Альфа-Бизнес Онлайн», подписав пакет документов.

Шаг 2. Подписать электронные формы:

  • согласия на подключение сервиса Alfa API (в интерфейсе системы "Альфа-Бизнес Онлайн"),
  • согласия на передачу персональных данных посредством Alfa ID (в интерфейсе системы "Альфа-Бизнес Онлайн"),
  • договора на информационно-технологическое обслуживание и приложений к нему (с помощью усиленной квалифицированной электронной подписи).

Комиссия за использование Alfa API составляет 2900 рублей за отчетный период (квартал).

Шаг 3. Ознакомиться с требованиями к TLS-сертификату.

Для использования методов API нужен сертификат, выпущенный Альфа-Банком. Для создания запроса на сертификат сертификата, необходимо пройти проверку системы Парнера API.

Шаг 4. Подготовить файлы конфигурации.

  • если вы используйте Mac OS, пример заполненного файла: Конфиг Mac openssl.cnf

  • если вы используете Windows, пример заполненного файла: Конфиг Windows openssl.cnf

Шаг 5. Создать и направить запрос на использование TLS-сертификата.

Для использования методов API нужен сертификат, выпущенный Банком. Для направления запроса на использование сертификата:

  • установите openssl на вашу платформу;

  • сгенерируйте закрытый ключ, введя команду в терминале:

openssl genrsa -des3 -out name.key 2048

  • на основе созданного ключа сгенерируйте запрос на сертификат (CSR) с приватным ключом RSA:

openssl req -new -key name.key -out name.csr

  • для проверки запроса на сертификат можно использовать команду в терминале:

openssl req -noout -text -in name.csr

  • отправьте запрос на сертификат на почтовый адрес baas@alfabank.ru с указанием темы письма "Запрос на сертификат". Никому не передавайте закрытый ключ. При компрометации закрытого ключа Партнер API несет ответственность за его компрометацию.

  • получите выпущенный сертификат, направленный с почтового адреса baas@alfabank.ru. Срок выпуска сертификата - 375 дней.
